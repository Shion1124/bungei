**デプロイコマンド（Predect関連）**

  

・これは毎日リセットして立ち上げる必要がある。（リセットしない場合、エラーが出てしまう）

  

  

まずはDockerdesktopの立ち上げ

  

**クラスターの起動**

minikube start

  

  

**クラスターの状態確認**

kubectl get nodes

  

  

**Prefectサービスの確認**

kubectl get svc -A | grep prefect

  

  

**Prefect2.xの立ち上げ（localhost:4200）**

uv run prefect server start

  

  

**Grafanaを確認**

kubectl get svc -n monitoring

  

  

**見当たらない場合**

kubectl get svc -A | grep grafana

  

  

**Grafanaの立ち上げ（localhost:3000）**

kubectl port-forward svc/prometheus-grafana 3000:80 -n monitoring

  

  

**Prometeusの立ち上げ（localhost:9090 ）**

kubectl port-forward svc/prometheus-kube-prometheus-prometheus 9090:9090 -n monitoring

  

  

**Alertmanagerの立ち上げ（localhost:9093）**

kubectl port-forward svc/alertmanager-operated 9093:9093 -n monitoring

  

  

  

**Prefect Exporterや独自Exporterのmetrics（localhost:8000）**

kubectl port-forward svc/prometheus-prefect-exporter 8000:80 -n monitoring

  

  

**PPOのOptuna最適化のための接続（**[**http://localhost:8001/docs**](http://localhost:8001/docs)**）**

uv run uvicorn src.api.ppo_api:app --host 0.0.0.0 --port 8001 --reload

  

  

**PPOの8080番ポートはメトリクス（監視・可視化用）専用のサブサーバー（localhost:8080）**

  

・PPOのOptuna最適化のための接続を行うことで自動起動します。






**PPOメトリクスの確認：PrometheusのGraph機能で以下を入力**


# PPOサービスの基本情報

python_info{job="ppo-agent-service"}

  

# PPOサービスのガベージコレクション統計

python_gc_collections_total{job="ppo-agent-service"}

  

# PPOサービスのメモリ使用量

python_gc_objects_collected_total{job="ppo-agent-service"}

  

# PPOサービスのすべてのメトリクス

{job="ppo-agent-service"}

  

  

**PPOメトリクスの確認方法**

  

# PPO固有メトリクスの確認

curl -s http://localhost:8080/metrics | grep -E "ppo_"

  

# PPOメトリクスサマリーの確認

curl -H "Authorization: Bearer dev-token-123" http://localhost:8001/metrics/summary

  

# PPOヘルスチェック

curl http://localhost:8001/health

  

  

**アクセス可能なmetricsエンドポイント例**

- svc/prometheus-prefect-exporter（80/TCP）
- svc/ppo-agent-service（8000/TCP, /metrics）
- svc/meta-model-service（8001/TCP, /metrics）
- svc/trade-executor-service（8002/TCP, /metrics）
- svc/streamlit-monitor-service（8501/TCP, /metrics）
- その他、metrics.pyやenhanced_metrics.pyで起動するExporter（デフォルト8080や8001など）
