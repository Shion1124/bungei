## ファイルの保管場所
 - スクリプト毎にファイルがどこにあるのかを各Phaseごとのファイルに全て記述します。

```bash
mkdir -p src/data_pipeline src/learning_pipeline/environments src/learning_pipeline/agents src/learning_pipeline/trainers src/backtesting_pipeline
```
```bash
mkdir -p scripts notebooks tests/unit tests/integration tests/e2e
```
Phase1・Phase2の各工程ファイルに、  
「この工程で作成・管理すべきスクリプト/モジュールの分類・保存先ディレクトリ」を明記したバックアップ用文章を冒頭に追記しました。

例：  
- データ取得用スクリプト → `src/data_pipeline/collector.py`
- バックテストエンジン → `src/backtesting_pipeline/engine.py`
- テスト → `tests/unit/`
- 補助スクリプト → `scripts/` など

この分類指針をPhase3以降の全ファイルにも順次追記し、  
「どのスクリプト・モジュールがどこにあるか」を明確にし、  
今後の実装・管理・バックアップの一貫性を担保します。



Phase3〜Phase5の全ファイルにも、  
「この工程で作成・管理すべきスクリプト/モジュールの分類・保存先ディレクトリ」を明記したバックアップ用文章を冒頭に追記しました。

これにより、**各工程で作成・管理すべきファイルが「どのディレクトリに分類されるか」**が明確になり、  
今後の実装・管理・バックアップの一貫性が担保されます。

- 例：PPO+LSTMモデル → `src/learning_pipeline/agents/ppo_agent.py`
- PER/HERバッファ → `src/learning_pipeline/agents/memory.py`
- Optuna最適化 → `src/learning_pipeline/trainers/optimizer.py`
- 本番用Dockerfile → `src/各モジュール直下` など

---

今後はこの分類指針に沿って、  
- 新規スクリプト・モジュールの作成
- テスト・設定ファイルの管理
- バックアップ・ドキュメントの整理

を進めていきます。
