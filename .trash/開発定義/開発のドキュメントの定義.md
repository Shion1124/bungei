## 開発に関するドキュメントの定義

この開発に関するドキュメントの定義は要件定義・設計書に沿った内容ではあるが、更にその内容の内部を作業するにあたってのドキュメントである。つまり、作業一つ一つを行うにあたりAIエージェントが読解しやすい内容を設計書に沿った作業工程ごとにあらためて定義し、作業を進めていくためのものである。

1. 作業工程に関する全容を進めるポイントや進め方などを具体的に記載する
2. その内容を細分化し、作業工程を優先順位に沿って進める内容を記載する
3. その内容に沿って作業を進める
4. 作業完了時には必ず、設計書やReadme.mdなどのドキュメントに作業内容を記述する。
5. 知見管理システムのデバッグフォルダのファイルに行った作業内容を記述する。
6. git commitを行う。

### 注意点
- Corsorで作業を進める場合は、作業工程の内容にもよるが、一つの作業をできるだけ細分化させることにより、AIエージェントのパフォーマンスが良くなる。

- 現在の作業の位置付けが作業の全容の中でどの部分が完了して、どの部分が未完了かが一目瞭然になることでAIエージェントは迷いなく作業を完遂することができるようだ。

- 1~3の内容はmdファイルに記載してdocディレクトリに保存する必要がある。
